# Multi-Model Compatibility Test Strategy

## 测试目标

验证 Claude 和本地模型（如 Qwen）之间的输出结构兼容性。

## 测试用例

### TC-001: Tool Calling Schema 兼容性
- **方法**: 同一 Tool Schema，分别用 Claude 和 Qwen 调用
- **通过标准**: 两者输出的 JSON 结构一致（字段名、类型）

### TC-002: Prompt 兼容性测试
- **方法**: 同一 Prompt，分别用两个模型执行
- **通过标准**: 两者都能正确理解并执行

### TC-003: Fallback 切换测试
- **方法**: 主模型不可用时自动切换到备用模型
- **通过标准**: 切换过程无数据丢失，输出格式一致

### TC-004: 输出质量对比
- **方法**: 同一组问题，分别用两个模型回答
- **通过标准**: 质量评分差距 < 10%

## 模型配置

| 模型 | 角色 | 优先级 |
|------|------|--------|
| claude-opus-4-7 | 主模型 | 1 |
| qwen-xxx | 备用模型 | 2 |

## 兼容性要求

- Tool Calling JSON 结构必须完全兼容
- Prompt 不得使用模型特有语法
- 输出解析器必须兼容两种模型的输出格式
