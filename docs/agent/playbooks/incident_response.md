# Incident Response Playbook

> Agent 系统故障响应手册。

## 故障等级

| 等级 | 定义 | 响应时间 |
|------|------|---------|
| P0 | 系统不可用 | 立即 |
| P1 | 核心功能受损 | 1 小时内 |
| P2 | 非核心功能异常 | 4 小时内 |
| P3 | 轻微问题 | 下个工作日 |

## P0: 系统不可用

### 可能原因
- Claude API 不可用
- 数据库连接失败
- Redis 连接失败

### 排查步骤
1. 检查各服务健康状态
2. 查看日志中的错误信息
3. 检查网络连接

### 临时缓解
- 切换到备用 LLM（见 `local_llm_switch.md`）
- 重启失败的服务

## P1: 核心功能受损

### 可能原因
- Tool Calling 成功率下降
- Context 遗忘严重
- 响应延迟过高

### 排查步骤
1. 运行 `tests/ai_runtime/` 测试套件
2. 检查最近的 Prompt 或代码变更
3. 查看 `known_issues.md` 是否有相关问题

### 临时缓解
- 回退最近的变更
- 降低并发量

## P2/P3: 非核心/轻微问题

### 排查步骤
1. 记录问题到 `known_issues.md`
2. 分析根因
3. 制定修复计划

## 故障复盘

每次 P0/P1 故障后必须：
1. 记录到 `memory/failed_attempts.md`
2. 更新 `playbooks/` 中的相关手册
3. 评估是否需要新增测试用例
