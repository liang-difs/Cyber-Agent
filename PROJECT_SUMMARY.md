## 一、项目概述

CyberSec Agent 是一个**基于大语言模型（LLM）驱动的通用网络安全智能体**，具备自主任务理解、多场景决策执行、可解释推理链路和自动化响应能力。系统采用 ReAct（Reasoning + Acting）推理框架，集成 23 个安全工具、多智能体协同引擎、知识图谱、Sigma/YARA 规则引擎和 RAG 检索增强生成，覆盖渗透测试、应急响应、威胁狩猎、漏洞评估、恶意软件分析和逆向工程六大安全场景。

### 核心能力

| 能力 | 说明 |
|------|------|
| **自主任务理解** | 自动识别 10+ 种输入格式（日志/PCAP/压缩包/二进制/配置文件/API 文档），路由到正确的工具链 |
| **多场景决策执行** | 6 种预定义任务模板 + LLM 动态规划，支持条件分支和策略切换 |
| **可解释推理** | 全链路 DecisionTracker 记录 Thought → Action → Observation → Final Answer，支持审计报告导出 |
| **自动化响应** | 5 种响应动作（IP 封禁/主机隔离/通知/文件隔离/账户禁用），支持回滚 |
| **知识增强** | RAG 混合检索（ChromaDB + BM25 + RRF 融合）+ 知识图谱（实体/关系/路径查询） |

## 五、核心贡献点

### 5.1 自研 ReAct Agent 框架（非 LangChain）

完全自主实现的 ReAct 推理循环，不依赖 LangChain 等第三方 Agent 框架：

- **5 级 JSON 解析器**：容错处理 LLM 输出中的格式错误（markdown 包裹、尾逗号、注释等）
- **上下文压缩器**：按工具类型差异化压缩（PCAP 保留完整异常列表、CVE 保留统计数据）
- **输入格式检测器**：三级优先级（魔数 > 扩展名 > 内容启发式），自动路由到正确的工具链
- **工具参数别名映射**：自动纠正 LLM 输出的参数名拼写错误

### 5.2 多智能体协同引擎

5 种角色的 Agent 协同工作：

| 角色 | 职责 | 能力 |
|------|------|------|
| Coordinator | 任务分解、调度、聚合 | 依赖感知执行、并行步骤、死锁检测 |
| Planner | 执行计划生成 | 6 种场景模板 + LLM 动态规划 |
| Analyzer | 数据分析 | 日志分析、流量分析、异常检测、IoC 提取 |
| Executor | 工具执行 | 端口扫描、漏洞扫描、服务枚举 |
| Result | 结果生成 | 报告生成、建议输出 |

**创新点**：依赖感知的步骤执行引擎，上游步骤的结果自动注入下游参数（IoC、漏洞、端口、服务等）。

### 5.3 知识图谱与威胁情报关联

自研内存图数据库，支持：

- **24 种实体类型**：CVE、恶意软件、威胁行为者、ATT&CK 技术/战术、IP、域名、哈希等
- **30 种关系类型**：利用、归因、投递、通信、缓解等
- **图算法**：BFS 邻居遍历、最短路径查找、子图提取
- **实体提取器**：正则 + NLP 从文本中自动提取 IoC 和 ATT&CK 实体

### 5.4 Sigma/YARA 规则引擎

- **Sigma 引擎**：YAML 解析 + 条件评估（AND/OR/NOT）+ 日志源匹配 + 置信度计算
- **YARA 引擎**：规则编译 + 文件/数据匹配 + 元数据提取
- **统一管理**：RuleManager 提供统一接口，支持日志匹配、文件匹配、数据匹配三种模式

### 5.5 端到端可解释性

DecisionTracker 记录完整的推理链路：

```
用户查询 → Thought(分析思路) → Action(工具调用) → Observation(工具返回) → ... → Final Answer
   │           │                    │                     │                        │
   │        confidence           tool_name             confidence              confidence
   │        evidence             tool_input            evidence                evidence
   │                                                                              
   └── trace_id / session_id / tenant_id / timestamps / duration_ms
```

支持 JSON/Markdown 格式的审计报告导出。

## 六、创新点

### 6.1 多模态输入自动识别

不同于传统的"用户手动指定工具"模式，系统通过三级检测自动识别输入格式：

1. **文件魔数检测**（置信度 0.95）：读取文件头 16 字节，匹配 pcap/zip/exe/elf 等格式
2. **扩展名映射**（置信度 0.80）：30+ 种扩展名到工具链的映射
3. **内容启发式**（置信度 0.70-0.90）：正则匹配日志格式、Swagger 关键字、IoC 模式、Sigma/YARA 语法

### 6.2 LLM 驱动的动态任务规划

通用任务不再走硬编码模板，而是调用 LLM 根据任务描述动态生成 2-6 步执行计划：

```python
# 传统方式：固定模板
if task_type == "incident_response":
    steps = [analyze, extract_ioc, lookup_threat, report]  # 永远相同

# 创新方式：LLM 动态规划
prompt = f"任务: {task.description}\n可用工具: {...}\n生成最优计划"
steps = await llm.complete(prompt)  # 根据具体场景生成
```

### 6.3 工具链自动编排

输入检测器返回推荐的工具链序列，Agent 按链执行：

```
输入: /data/capture.pcap
  → 检测: pcap (magic bytes, conf=0.95)
  → 工具链: pcap_analysis → ip_threat_analysis → ioc_lookup
  → Agent 自动串联执行
```

### 6.4 响应动作闭环

从检测到响应的完整闭环：

```
告警产生 → Sigma 规则匹配 → 知识图谱更新 → 响应动作触发 → 多智能体分析
                                                    ↓
                                            BlockIP / IsolateHost / Notify
                                                    ↓
                                              数据库持久化 + 回滚支持
```

### 6.5 多引擎搜索融合

RAG 检索采用 Reciprocal Rank Fusion（RRF）融合向量搜索和关键词搜索：

```
ChromaDB 向量搜索 (BGE-M3 嵌入, 余弦相似度)
        ↓                    ↓
    排名列表 1            排名列表 2
        ↓                    ↓
        └────── RRF 融合 ──────┘
                  ↓
           Top-K 结果返回
```
